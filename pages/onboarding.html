MCSfMSC Onboarding

Complete guide for onboarding new MCSfMSC customers — from initial request to active service delivery

Owner: Shalini Hada Last Updated: February 25, 2025

Customer Onboarding Flow

Follow these steps to onboard a new MCSfMSC customer end-to-end

1

Step 1: Customer Request & Intake

A new MCSfMSC engagement is initiated when a customer request is received through the sales team or account team.

Actions Required:

  • Receive customer request via Account Team / CSAM
  • Validate customer has appropriate licensing (M365 E5 Security, EMS E5, or standalone plans)
  • Confirm MCSfMSC service SKU is included in the customer's contract
  • Create engagement record in the tracking system
  • Assign Service Delivery Manager (SDM)

Customer Setup Checklist on SharePoint
3

Step 3: Access Provisioning

Set up the necessary access permissions for the MCSfMSC team to support the customer.

Access Requirements:

  • Azure Lighthouse: Configure delegated access for multi-tenant management
  • GDAP: Set up Granular Delegated Admin Privileges for partner access
  • Entra ID Roles: Security Reader, Security Operator (minimum required roles)
  • Sentinel Roles: Microsoft Sentinel Responder / Contributor
  • Defender Portal: Security Administrator role for configuration
  • Teams Access: Guest access to customer collaboration channels

Access Governance:

  • All access must follow least-privilege principle
  • Use PIM for elevated access with time-bound activation
  • Quarterly access reviews required
  • Access must be revoked within 24 hours of engagement end
4

Step 4: Service Configuration

Configure the security services based on customer requirements and the engagement scope.

Configuration Activities:

  • Sentinel Setup: Workspace creation, data connectors, analytics rules, automation playbooks
  • Defender XDR: Device onboarding, ASR rules, alert tuning, device groups
  • Entra ID Security: Conditional Access policies, Identity Protection, PIM configuration
  • Defender for Cloud: Enable plans, configure CSPM, set up alerts
  • MDO: Safe Links, Safe Attachments, anti-phishing policies
  • Purview: Sensitivity labels, DLP policies, insider risk
View Configuration Playbooks
5

Step 5: Kickoff & Handoff to Delivery

Conduct the formal customer kickoff and transition from onboarding to ongoing delivery.

Kickoff Meeting Agenda:

  1. Team introductions (MCSfMSC team ↔ Customer team)
  2. Engagement scope and objectives review
  3. Service delivery model explanation
  4. Communication plan (cadence, channels, escalation)
  5. Success criteria and KPI alignment
  6. Security posture baseline review
  7. First 30-day action plan
  8. Q&A

Handoff Deliverables:

  • Completed Customer Setup Checklist
  • Access provisioning confirmation
  • Initial service configuration summary
  • Engagement project plan
  • Communication matrix
