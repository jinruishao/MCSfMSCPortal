Issue Resolution

Guided workflows, escalation paths, and resolution playbooks for Microsoft Security Cloud issues

Owner: Curtis Lewis Last Updated: October 31, 2025

Severity Classification

Classify issues by severity to determine the correct escalation path and SLA

Sev 1

Critical

Complete service outage or security breach impacting production

Response Time: 15 minutes
Resolution Target: 4 hours
Escalation: Immediate to Sev 1 Manager
Communication: Every 30 minutes

Examples:

  • Microsoft Sentinel completely down in production
  • Active security breach detected
  • All Defender alerts stopped processing
  • Entra ID authentication outage
  • Customer data exposure incident
Sev 2

High

Major functionality degraded, workaround may exist

Response Time: 1 hour
Resolution Target: 8 hours
Escalation: Service Delivery Manager
Communication: Every 2 hours

Examples:

  • Sentinel data ingestion delayed > 1 hour
  • Defender for Endpoint partial deployment failure
  • Conditional Access policies not applying correctly
  • DLP policies failing to trigger
  • Multiple failed sign-in alerts not generated
Sev 3

Medium

Non-critical functionality impaired, standard SLA

Response Time: 4 hours
Resolution Target: 3 business days
Escalation: Support Engineer
Communication: Daily update

Examples:

  • Dashboard widgets not loading correctly
  • Minor configuration drift detected
  • Report generation intermittently failing
  • Non-critical alert rule needs tuning
  • User provisioning delays
Sev 4

Low

Cosmetic issues, feature requests, general questions

Response Time: 1 business day
Resolution Target: 5 business days
Escalation: Queue-based
Communication: As needed

Examples:

  • Documentation update request
  • Feature enhancement request
  • Training or how-to questions
  • Minor UI display issues
  • Best practice guidance requests

Escalation Path

Follow the structured escalation process based on issue severity

Level 1: Frontline Support

Initial triage and basic troubleshooting. Check known issues database, verify configurations.

Microsoft Sentinel Issues 12 articles

Data connector not ingesting logs

Symptoms: Log data from Azure AD, Office 365, or third-party sources stops appearing in Sentinel.

Resolution Steps:

  1. Check data connector health status in Sentinel > Data connectors
  2. Verify API permissions and credentials haven't expired
  3. Check Azure Monitor Agent (AMA) deployment status
  4. Review DCR (Data Collection Rules) for correct configuration
  5. Verify Log Analytics workspace hasn't reached capacity limits
  6. Check for any Azure service health advisories

Analytics rule not triggering alerts

Symptoms: Scheduled or NRT analytics rules not generating incidents.

Resolution Steps:

  1. Verify the rule is enabled and check last run time
  2. Run the KQL query manually in Log Analytics to verify results
  3. Check query time range and entity mapping configuration
  4. Review alert threshold settings
  5. Ensure the data tables referenced in query are being populated

High data ingestion costs

Symptoms: Unexpected cost increases in Log Analytics/Sentinel workspace.

Resolution Steps:

  1. Run usage analysis workbooks to identify top data sources
  2. Review basic/analytics log tiers for appropriate data classification
  3. Implement data transformation rules to reduce noise
  4. Configure data collection rules (DCR) to filter at source
  5. Consider commitment tiers for predictable pricing
Microsoft Defender XDR Issues 10 articles

Defender for Endpoint onboarding failures

Symptoms: Devices not appearing in MDE portal after onboarding package deployment.

Resolution Steps:

  1. Verify device meets minimum OS requirements
  2. Check network connectivity to MDE cloud service URLs
  3. Verify onboarding package is for the correct tenant and OS
  4. Check Windows Event Logs for onboarding errors (Event ID 47, 48)
  5. Run MDE Client Analyzer tool for comprehensive diagnostics
  6. Ensure no conflicting AV/EDR products are installed

Automated investigation stuck or failed

Symptoms: AIR investigations not completing or showing errors.

Resolution Steps:

  1. Check the investigation graph for specific failed actions
  2. Verify automation level settings in device groups
  3. Ensure necessary permissions (Security Admin role)
  4. Check if device is reachable and responsive
  5. Review pending actions that require manual approval

Alert suppression not working

Symptoms: Suppression rules not filtering known false positives.

Resolution Steps:

  1. Verify suppression rule scope (org-wide vs. device group)
  2. Check IOC conditions match exactly (case sensitivity)
  3. Review rule expiration date settings
  4. Ensure the alert title/detection source matches the rule filter
Microsoft Entra ID Issues 8 articles

Conditional Access policies blocking legitimate users

Symptoms: Users unable to sign in due to overly restrictive CA policies.

Resolution Steps:

  1. Use "What If" tool in Entra ID > Conditional Access to simulate
  2. Check Sign-in logs for specific CA policy that blocked the user
  3. Review policy assignments (include/exclude groups)
  4. Verify named locations and trusted IP ranges
  5. Check for conflicting policies with different grant controls
  6. Use Report-only mode for testing before enforcement

PIM role activation failures

Symptoms: Users unable to activate eligible roles through Privileged Identity Management.

Resolution Steps:

  1. Verify the user has an eligible role assignment (not expired)
  2. Check if MFA is required and user has registered MFA methods
  3. Review approval workflow — may need approver action
  4. Check maximum activation duration settings
  5. Verify no conflicting Conditional Access policies for PIM
Microsoft Purview Issues 6 articles

DLP policies not detecting sensitive data

Symptoms: DLP policies failing to identify and protect sensitive information.

Resolution Steps:

  1. Verify policy is enabled and targeting correct locations (Exchange, SharePoint, OneDrive, Teams)
  2. Check sensitive information type (SIT) definitions and confidence levels
  3. Test with DLP policy test mode using sample data
  4. Ensure Unified Labeling client is installed on endpoints
  5. Review DLP activity explorer for policy match events
  6. Check if exact data match (EDM) tables are properly uploaded

Sensitivity labels not applying

Symptoms: Auto-labeling or manual labeling not working in Office apps or SharePoint.

Resolution Steps:

  1. Verify label policy is published and scoped correctly
  2. Check Office app version supports built-in labeling
  3. Remove any legacy AIP add-in if present
  4. Allow 24h for label policy propagation
  5. Check Content Explorer for simulated label application
Defender for Office 365 Issues 5 articles

Phishing emails bypassing Safe Links/Safe Attachments

Symptoms: Users receiving phishing emails that were not caught by MDO.

Resolution Steps:

  1. Check the email in Threat Explorer for delivery reason
  2. Review Safe Links/Safe Attachments policy scope
  3. Check for transport rules that may be bypassing scanning
  4. Verify no IP/sender overrides in allow lists
  5. Submit the email to Microsoft as a false negative
  6. Review and tighten anti-phishing policies
